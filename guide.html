Pubs

What is a pub?

A Pub can be though of similarly to a git repo with added metadata. At it's base, it is a collection of versioned files. Added metadata enable us to provide titles, discussions, permissions, and forking.

A Pub is a container that includes a file or collection of files. The contents of this container render out to a document. A Pub has a title, associated contributors, discussions, and versions.

At it's simplest, a Pub can replicate a simple traditional document or PDF. With more effort, a Pub can become an interactive, rich, complete research documentation and communication tool.

Pubs (and thus the contents within them) can be cloned, forked, and reused. Pubs can be collaboratively updated, organized by different labels, and curated by Journals.

How do I create a Pub

Creating a pub creates a new row in the Pub table, new collaborator(s). Pubs are private by default. Pubs are initialized with no versions, but tools for importing content or opening the editor are made immediately available.

To create a Pub, click the 'New Pub' button in the top-right corner of the site. (TODO: show image of button). This button will lead you to a page that lets you specify the title, etc of a Pub (TODO: specify what fields will be present - show form. Once your Pub has been created, you can add collaborators, create new versions by importing, open in an editor, upload files, etc.

What are versions?

Versions have their own table. They include a version message and other metadata. A version is linked to a specific Pub. There is a table that links a specific versions to specific files. Files can be identical across versions, but each version represents a unique set of files.

A version is a snapshot of the files within a Pub at a given time. Versions allow you to track the history of the pub as it evolves.

Versions can be public or private. Private versions are only visible to those listed as collaborators on a pub. Public versions are open to the public and are archived. Once made public, a version cannot be deleted as it is archived as an open publication.

How do I publish a version?

Navigate to the 'Versions' tab of your pub. To make a version public, click on the 'Publish Version' button. Publishing is a permanent act (imagine it like publishing a newspaper, there is no taking it back). Once published, the version and pub are archived. You can create as many versions as you like.

Can I delete after I publish

Once published, a version cannot be made private. A version can be redacted if situations prove it necessary, but the archive of the public work will remain.

Can I publish a clone of a private pub?

If you have access to a private pub, and clone the pub to suggest edits or drive the document in a different direction, you cannot publicly publish any ancestors of that work until the root parent which was cloned is made publicly available.

How do we show diffs? What are we diffing?

The difference between any to snapshots of a given file can be calculated. For many file types, we will provide a visual difference that can be easily compared. For binary or proprietary files, a visual diff is not possible, so the two files are simply presented.

How is the document file stored (xml)

All document files are stored as XML documents. PubPub XML documents use schema standards to structure the document so that it is parsable by machines and can be rendered into many different formats.
